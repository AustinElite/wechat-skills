# wechat-topic-planner

## 作用
用于微信公众号的选题策划、角度拆分、栏目化设计与候选题池生成。

## 何时使用
当用户出现以下意图时使用：

- 想公众号选题
- 做内容策划
- 给一个热点找切入角度
- 围绕某个主题列出多篇文章方向
- 想做栏目但不知道怎么起盘

## 输入优先级
优先从用户提供的信息中提取：

1. 账号定位
2. 目标读者
3. 行业或主题范围
4. 写作目标
5. 是否追求热点、深度、转化或个人表达

如果信息不完整，不要先追问一长串问题，先基于已有信息给出一版可执行方案，并明确你的假设。

## 工作流程
1. 先识别账号定位与目标读者。
2. 判断这次更适合热点型、常青型、观点型、案例型、教程型还是人物型选题。
3. 给出 6-12 个候选选题，不要只有标题，要说明切入角度。
4. 对每个选题补充：
   - 为什么值得写
   - 适合什么读者点开
   - 最佳文章结构建议
5. 如果用户要长期运营，再额外输出栏目化建议：
   - 栏目名称
   - 更新频率
   - 每个栏目的内容边界

## 输出格式
默认输出为以下结构：

### 1. 策划判断
- 账号定位：
- 目标读者：
- 本轮内容目标：
- 我采用的选题策略：

### 2. 候选选题
每个选题包含：
- 题目
- 核心切口
- 为什么会有人看
- 建议结构

### 3. 推荐优先级
- 最值得先写的 3 个题
- 每个题适合的发布时间或场景

## 质量要求
- 选题不能只有“正确”，还要有传播抓手
- 避免空泛大词，要尽量具体到冲突、变化、代价、案例或结果
- 尽量兼顾可读性与账号定位，不为了爆而失真
- 如果主题已经很卷，要主动提出差异化角度

## 禁止事项
- 不要输出一堆只有标题没有解释的流水账
- 不要把短视频标题逻辑生硬搬到公众号长文
- 不要默认所有文章都追求“爆款”，要尊重用户的表达目标
